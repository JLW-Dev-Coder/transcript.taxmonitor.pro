IRS Code 768 Meaning

IRS Code 768 is the Earned Income Credit posting entry. Each 768 line shows that EIC has been applied to the taxpayer's account and contributes to refund or balance-due outcomes. This page shows you how to track 768 through return processing, pair it with other credits, and explain EIC posting and refund timing to clients—without guesswork.

Plain-English definition

IRS Code 768 means the IRS posted the Earned Income Credit (EIC) to the account. It’s a credit entry that reduces tax and can increase the refund shown later on the transcript.

When does Code 768 appear?

  1. Return processing (most common) — The IRS accepts the EIC claim as part of the filed return and posts 768 during initial processing.
  2. Adjusted EIC amount — If the IRS changes credits during a math error correction or review, a later posting can change the effective EIC outcome.
  3. Refund timing context — 768 confirms the credit posted, but the refund path is confirmed by other codes (especially 846).

What does Code 768 usually signal?

  • EIC was applied to the module and is part of the refund/balance math.
  • It commonly pairs with other credits like 766 and posts around the same time as 150.
  • It does not guarantee immediate refund release; watch for holds, offsets, or other action codes.

What should a tax professional do next?

  1. Reconcile to the return — Confirm the posted EIC aligns to what was filed and what the client expects.
  2. Track the full refund sequence — Follow 768 + related credits through to 846 (refund issued) or any offset/hold codes.
  3. Set expectations — Explain that 768 is a credit posting, while the refund release is confirmed by later transcript activity.

FAQ: IRS Code 768

What does IRS Code 768 mean on an IRS transcript?

Code 768 means Earned Income Credit has posted to the taxpayer's account as part of processing the return.

Does Code 768 guarantee a refund?

No. Code 768 indicates that Earned Income Credit was applied, but refunds can still be delayed or reduced by review holds, refund freezes, or offsets.

Why do 768 and 766 often show the same posting date?

Because the IRS posts Earned Income Credit (768) and related credits (766) during initial return processing, it is common for both codes to share the Code 150 posting date.

How should I explain EIC posting vs. refund timing to a client?

Explain that Code 768 confirms the Earned Income Credit was applied, but the actual refund timing depends on additional checks, potential freezes, and offsets, which are reflected by other transaction codes on the transcript.
